# ADR 0065: README and thesis positioning

## Status

Accepted

## Context

The project now includes a complete regime-aware portfolio research workflow, advanced analytics, CLI workflows, and exportable research packages.

The README should communicate the project as a professional quantitative research engine rather than a collection of isolated modules.

The project also needs a clear thesis analysis document that explains the research motivation, methodology, and investment interpretation.

## Decision

Add a professional README and thesis analysis document.

The README will include:

- Executive summary
- Core research thesis
- System architecture diagram
- End-to-end workflow
- Key features
- Quickstart
- CLI examples
- Repository structure
- Methodology overview
- Limitations
- Roadmap

The thesis analysis document will explain the research question, motivation, methodology, interpretation, and future improvements.

## Consequences

The project is easier to understand, review, and present.

A recruiter, professor, or portfolio reviewer can quickly understand what the system does, why it matters, and how to run it.

Future documentation can add screenshots, rendered memo examples, and notebook walkthroughs.
